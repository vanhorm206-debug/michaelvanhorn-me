m365-bridge is an MCP server I built to give an AI coding agent live, read-and-write access to Outlook, Teams, and Calendar. It is in daily production use in my work.

Most AI tools can talk about your email; this one works in it: read the thread, draft the reply, check the calendar, through one governed interface. The hard part was not the connection, it was doing it safely, so the agent has exactly the access it needs and no more.

It reflects a larger conviction: AI is most useful to non-technical teams when it can reach into the tools they already live in, carefully.
